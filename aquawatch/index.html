This system follows a 4-layer IoT architecture: physical sensing → edge processing → cloud storage → user visualization. Data flows from hardware sensors through a NodeMCU ESP8266 microcontroller via WiFi to the Blynk IoT cloud platform, then to mobile/web dashboards.

1
Sensing Layer
Physical sensors collect water quality parameters continuously
Moisture FC-28TurbidityDS18B20TDS Gravity
2
Processing Layer
Microcontroller reads, processes and transmits sensor data
NodeMCU ESP8266Arduino IDEADC Conversion
3
Network & Cloud
WiFi transmits data to cloud for storage and processing
WiFi 802.11Blynk IoTMQTT Protocol
4
Application Layer
Users monitor data and receive alerts via dashboard
Web DashboardMobile AppPush Alerts

Project Objectives
1
Understand water quality parameters and their impact on human health and environment
2
Gain practical knowledge of IoT architecture including sensors, microcontrollers, and cloud platforms
3
Design and implement a real-time water quality monitoring system using IoT technologies
4
Learn sensor interfacing and data acquisition for pH, turbidity, temperature, dissolved oxygen
5
Analyze and visualize real-time data on a web dashboard with historical trends and alerts
